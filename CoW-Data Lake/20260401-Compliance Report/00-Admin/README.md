# 00 — Admin

**Purpose:** Ongoing project management documents maintained throughout the entire engagement. This folder is the operational backbone of the project — not a deliverable, but essential for keeping the engagement on track.

---

## Contents

| File | Description | Owner |
|---|---|---|
| `Project-Charter.md` | Engagement scope, objectives, team, and governance structure | Project Lead |
| `RACI-Matrix.md` | Who is Responsible, Accountable, Consulted, and Informed for each workstream | Project Lead |
| `Risk-Register.md` | Known risks, likelihood, impact, and mitigation actions — updated weekly | Project Lead |
| `Budget-and-Hours-Tracker.xlsx` | Planned vs. actual hours by team member and phase | Project Lead |
| `Change-Log.md` | Record of all scope, timeline, or deliverable changes with date and approval | Project Lead |

---

## When to Update

- **Project Charter** — Set at kickoff; update only if scope changes
- **RACI Matrix** — Finalize by end of Week 1; update if roles change
- **Risk Register** — Review and update every week
- **Budget Tracker** — Update hours at the end of each work week
- **Change Log** — Update immediately when any change is agreed with the client

---

## Naming Convention

```
YYYYMMDD-DocumentName-vN.md
```

Example: `20260407-Project-Charter-v1.md`

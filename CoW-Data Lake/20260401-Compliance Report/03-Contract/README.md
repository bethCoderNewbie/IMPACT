# 03 — Contract

**Purpose:** All legally binding agreements and formal authorizations for this engagement. Nothing in this folder should be edited after signing — treat all signed documents as permanent records.

---

## Contents

| File | Description | Status |
|---|---|---|
| `SOW-Draft.docx` | Working draft of the Statement of Work before client review | Draft |
| `SOW-Signed.pdf` | Countersigned final Statement of Work — authorizes the engagement | Signed |
| `NDA.pdf` | Non-disclosure agreement protecting CoW's confidential information | Signed |
| `Change-Orders/` | Subfolder for any approved amendments to the original SOW | Per change |

---

## What the SOW Must Cover

1. **Engagement Overview** — Purpose and context
2. **Scope of Services** — Exactly what will and will not be done
3. **Deliverables** — What the client receives, in what format, by when
4. **Timeline & Milestones** — Key dates and phase gates
5. **Fees & Payment Terms** — Total price, invoicing schedule, late payment terms
6. **Revision Policy** — How many rounds of revisions are included
7. **Acceptance Criteria** — How the client formally signs off on deliverables
8. **Confidentiality** — Data handling and non-disclosure obligations
9. **Termination Clause** — Conditions under which either party may exit
10. **Governing Law** — Jurisdiction (State of Kansas)

## Change Order Process

Any change to scope, timeline, or price after SOW signing requires a written Change Order. File signed change orders in `Change-Orders/` with the naming format:

```
YYYYMMDD-Change-Order-N-Description.pdf
```

Example: `20260415-Change-Order-1-Add-Training-Module.pdf`

## Slide Deck Structure (Standard)

Each deck should follow this structure:

1. **Title slide** — Project name, client, date, presenter
2. **Agenda** — What will be covered today
3. **Context recap** — Where we are in the engagement
4. **Key findings / content** — The substance of the session
5. **Recommendations or next steps** — What happens after this meeting
6. **Q&A / open discussion**
7. **Appendix** — Supporting detail for questions that arise

## Notes

- Finalize decks at least **24 hours before** each session
- Send agenda to client contact at the same time
- Archive previous versions in place (do not delete — rename with `_old` suffix if superseded)
